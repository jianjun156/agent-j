1. Claude Code 是什么

Claude Code 是 Anthropic 推出的 AI 编程代理，运行在你的终端里，能直接读写文件、执行命令、搜索代码库。它不是补全插件，而是一个能独立完成多步骤开发任务的自主代理。

支持环境：

  • 终端 CLI — 最核心的使用方式
  • VS Code / Cursor — 编辑器扩展
  • JetBrains — IntelliJ / PyCharm / WebStorm 插件
  • Desktop App — macOS / Windows 桌面端
  • GitHub Actions / GitLab CI — CI/CD 集成
  • Slack — @Claude 在团队频道执行编码任务

2. 安装与启动

# 安装（需要 Node.js 18+）
npm install -g @anthropic-ai/claude-code

# 启动交互模式
cd your-project
claude

# 带初始提示启动
claude "帮我看看这个项目的结构"

# 继续上次会话
claude --continue
# 或简写
claude -c

# 非交互模式（用于脚本/CI）
claude -p "修复所有 TypeScript 错误" --output-format json
第一次使用？在项目根目录运行 claude /init，它会自动生成一份 CLAUDE.md 帮你开始。

3. 记忆系统 — CLAUDE.md

一份好的 CLAUDE.md 是提升 Claude Code 输出质量的最有效手段。它是 Claude 每次启动时读取的"记忆文件"。

文件加载规则

Claude Code 启动时会向上遍历目录树，加载路径上所有 CLAUDE.md：

/mymonorepo/
├── CLAUDE.md        ← 启动时立即加载（根级指令）
├── frontend/
│   └── CLAUDE.md    ← 懒加载（访问 frontend/ 文件时才加载）
├── backend/
│   └── CLAUDE.md    ← 懒加载
└── api/
    └── CLAUDE.md    ← 懒加载

三条核心规则：

  1. 祖先目录：启动时立即加载（向上遍历）
  2. 子目录：懒加载（访问文件时才加载）
  3. 兄弟目录：永远不加载（在 frontend/ 不会加载 backend/CLAUDE.md）

CLAUDE.md 写什么

# CLAUDE.md

## 项目概述
这是一个 React + TypeScript 的电商前端，使用 Tailwind CSS。

## 代码规范
- 使用 TypeScript strict mode
- 组件用函数式写法 + hooks
- 文件命名用 kebab-case
- commit 信息格式：type(scope): description

## 常用命令
- `npm run dev` — 启动开发服务器
- `npm run test` — 运行测试
- `npm run lint` — 代码检查

## 项目结构
src/components/ — UI 组件
src/hooks/ — 自定义 hooks
src/api/ — API 请求层
src/types/ — TypeScript 类型定义

## 注意事项
- 不要修改 src/legacy/ 下的文件
- API 密钥在 .env.local 中，不要提交
个人偏好用 CLAUDE.local.md（加入 .gitignore），团队共享的放 CLAUDE.md。全局设置放 ~/.claude/CLAUDE.md。

4. 自定义命令（Commands）

Commands 是注入到当前上下文的 prompt 模板，用斜杠触发，适合重复性工作流。

创建命令

.claude/commands/
├── fix-bug.md      → /fix-bug
├── add-test.md     → /add-test
└── deploy.md       → /deploy 
# .claude/commands/fix-bug.md
---
description: 分析并修复指定的 bug
---

请按以下步骤修复 bug：

1. 先理解 bug 描述：$ARGUMENTS
2. 搜索相关代码，找到根因
3. 编写修复代码
4. 添加测试用例覆盖该 bug
5. 运行测试确认通过
6. 总结修改内容

使用：/fix-bug 用户头像上传后不显示

常用内建命令

  • /init — 初始化项目 CLAUDE.md
  • /plan — 进入计划模式（只读，不修改代码）
  • /diff — 查看未提交的改动
  • /context — 可视化当前上下文使用量
  • /cost — 查看当前会话 token 消耗
  • /model — 切换模型
  • /memory — 编辑 CLAUDE.md
  • /voice — 语音输入（支持 20 种语言）
  • /chrome — Chrome 浏览器自动化
  • /stats — 查看使用统计和连胜记录

5. 子代理（Subagents）

Subagents 是在隔离上下文中运行的自主代理，有自己的工具、权限、模型和记忆。

创建子代理

# .claude/agents/code-reviewer.md
---
name: code-reviewer
description: "PROACTIVELY review code changes for bugs and improvements"
model: sonnet
tools: Read, Glob, Grep, WebSearch
maxTurns: 10
---

你是一个代码审查专家。审查时关注：
1. 逻辑错误和边界情况
2. 性能问题
3. 安全漏洞
4. 代码可读性
5. 测试覆盖

子代理 vs 命令

特性 Commands Subagents
上下文 共享当前会话 完全隔离
工具权限 继承所有 可限制
模型 继承 可指定
适用场景 简单 prompt 模板 复杂自主任务

6 个内建子代理

  1. general-purpose — 默认通用代理（研究、搜索、自主开发）
  2. Explore — 快速代码搜索（haiku 模型，只读）
  3. Plan — 计划模式（只读，设计实现方案）
  4. Bash — 终端命令执行
  5. statusline-setup — 配置状态栏
  6. claude-code-guide — Claude Code 使用指南

6. 技能（Skills）

Skills 是可配置、可预加载、自动发现的知识模块，比 Commands 更强大。

.claude/skills/
└── deployment/
    └── SKILL.md    ← 技能定义

Skills 支持渐进式披露（progressive disclosure）——Claude 先看到技能描述，需要时才加载完整内容。

Anthropic 维护了一套官方 Skills 库，包括 /simplify（代码简化）和 /batch（批量操作）等内建技能。

7. MCP 服务器

MCP（Model Context Protocol）让 Claude Code 连接外部工具、数据库和 API。

// .mcp.json — 项目级 MCP 配置
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres"],
      "env": {
        "DATABASE_URL": "postgresql://localhost/mydb"
      }
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}

配置后 Claude 就能直接查询数据库、操作 GitHub Issues/PRs 等。

8. 工作流编排模式

高效使用 Claude Code 的核心模式：Command → Agent → Skill

所有主要工作流都收敛于同一个架构模式：
Research → Plan → Execute → Review → Ship

实例：天气编排器

# .claude/commands/weather-orchestrator.md
---
description: 完整的天气应用开发工作流
---

请按以下编排流程执行：

1. **研究**：用 Explore 代理搜索现有代码结构
2. **计划**：用 Plan 代理设计实现方案
3. **执行**：实现代码，写测试
4. **审查**：用 code-reviewer 代理审查改动
5. **交付**：运行测试，提交代码

并行开发：Git Worktrees

# 在隔离的 worktree 中启动 Claude
claude --worktree

# 或手动创建
git worktree add ../my-feature feature-branch
cd ../my-feature
claude

每个 Agent 有自己的工作副本，互不干扰。适合多 Agent 并行开发不同功能。

Agent Teams（实验性）

# 启用多 Agent 团队协作
CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1 claude

多个 Agent 在同一代码库并行工作，共享任务协调。

9. 配置系统

Claude Code 的配置分 5 层优先级（高到低）：

  1. Managed — 组织策略（IT 部署，不可覆盖）
  2. 命令行参数 — 当次会话生效
  3. .claude/settings.local.json — 个人项目配置（git-ignored）
  4. .claude/settings.json — 团队共享配置（提交到 git）
  5. ~/.claude/settings.json — 全局个人默认

常用配置示例

// .claude/settings.json
{
  "model": "opus",
  "language": "chinese",
  "alwaysThinkingEnabled": true,
  "permissions": {
    "allow": [
      "Read(**)",
      "Glob(**)",
      "Grep(**)"
    ],
    "deny": [
      "Bash(rm -rf *)"
    ]
  }
}

权限模式

  • default — 每次操作都询问
  • plan — 只读模式，不修改代码
  • acceptEdits — 自动接受文件编辑，其他操作仍询问
  • bypassPermissions — 跳过所有权限提示
  • auto（推荐）— AI 安全分类器自动判断，平衡效率和安全
bypassPermissions 会跳过所有安全检查，仅在完全信任的环境中使用。推荐用 auto 模式。

10. 实战技巧

💡 先计划再执行

复杂任务先进入计划模式：

claude /plan "重构用户认证模块，从 JWT 迁移到 OAuth2"

审核计划后再执行，避免大规模修改方向偏差。

💡 善用 Checkpointing

Claude Code 自动基于 git 跟踪每次编辑。搞砸了？双击 Esc 或 /rewind 回退。

💡 用 /context 监控 token

上下文快满时，开新会话比在旧会话里硬塞更高效。

💡 提示工程

  • 给具体目标，而非模糊指令（❌ "优化代码" → ✅ "把 getUserList 的 N+1 查询改为批量查询"）
  • 分步骤拆解复杂任务
  • 提供上下文："这个项目用 Express + Prisma + PostgreSQL"
  • 明确约束："不要修改 migration 文件"

💡 非交互模式用于 CI

# 在 CI 中自动修复 lint 错误
claude -p "修复所有 ESLint 错误" \
  --permission-mode auto \
  --max-turns 20 \
  --max-budget-usd 5

💡 Chrome 浏览器集成

# 启用 Chrome 自动化
claude --chrome
# 然后你可以说：
# "打开 localhost:3000 测试登录流程"
# "截图当前页面的 responsive 布局"

💡 语音输入

# 在会话中启用
/voice
# 按住快捷键说话，支持 20 种语言

💡 远程控制

# 在电脑上启动 Claude Code 后
/remote-control
# 然后用手机浏览器扫码继续操作
