从零打造你的专属 AI Agent（像 J 一样）

本指南面向企业微信用户，目标是在你自己的电脑上跑一个 24 小时在线的 AI Agent，通过企业微信随时对话。 整个过程大约需要 20-40 分钟。

你最终会得到什么？

  • 一个属于你自己的 AI Agent，有名字、有性格、有记忆
  • 通过企业微信随时跟它对话（支持文字、图片、语音、文件）
  • 它能帮你搜索、分析、写文档、建智能表格、定时提醒……
  • 每次对话它都会记住上下文，越用越懂你

前置准备

项目 说明
企业微信 安装最新版客户端，且有创建智能机器人的权限（管理员可配置）
一台电脑 Mac / Windows / Linux 都行，建议保持常开
Node.js >= 22 终端运行 node --version 检查，没有就 brew install node（Mac）或去 https://nodejs.org 下载
AI 模型 API Key 推荐 Anthropic Claude（https://console.anthropic.com/ 注册并创建 API Key）

💡 关于 API Key：这是让 AI "思考"的钥匙。Claude Sonnet 性价比最高（约 $3/百万 token），日常使用每天几毛到几块钱。

第一步：安装 OpenClaw（5 分钟）

OpenClaw 是 Agent 的运行平台，相当于给 AI 一个"身体"。

Mac / Linux：

curl -fsSL https://openclaw.ai/install.sh | bash

Windows（PowerShell）：

iwr -useb https://openclaw.ai/install.ps1 | iex

⚠️ Windows 强烈建议先安装 WSL2（Ubuntu），然后在 WSL 里执行 Linux 命令，兼容性更好。

安装完验证：

openclaw --version

第二步：初始化 OpenClaw（10 分钟）

openclaw onboard --install-daemon

向导会一步步引导你：

  1. 选择模式 → 选 Local（本地运行）
  2. 配置 AI 模型 → 选 Anthropic，粘贴 API Key，选默认模型（推荐 claude-sonnet-4-6）
  3. 配置 Gateway → 端口默认 18789，直接回车
  4. 渠道先跳过 → 企业微信我们用更简单的方式接入（见第三步）
  5. 安装后台服务 → 选 Yes，开机自启

跑完后检查：

openclaw gateway status   # 应该显示 running

🎉 此时你已经可以在浏览器里聊天了！ 打开 http://127.0.0.1:18789/ 试试。

第三步：接入企业微信（10 分钟）

📌 企业微信现在支持每个成员直接在客户端创建智能机器人，不需要管理员后台操作了！ 官方文档：https://open.work.weixin.qq.com/help/wap/detail?docid=21657

方式一：扫码一键接入（最简单 ⭐推荐）

  1. 安装企微插件，在终端执行： bash npx -y @wecom/wecom-openclaw-cli install

  2. 企微扫码接入：终端会显示二维码，用企业微信扫描

  3. 一键创建机器人：扫码后点击「一键创建机器人」

  4. 自动接入完成：机器人自动关联到你本地的 OpenClaw，点击「前往使用」就能开始对话了！

就这么简单，4 步搞定 🎉

方式二：手动配置（需要更多控制时）

如果你想自定义配置，或者在云服务器上部署：

  1. 打开企业微信客户端 → 工作台 → 智能机器人 → 创建机器人 → 手动创建
  2. 选择 API 模式
  3. 连接方式选 「使用长连接」（无需域名/IP，推荐！）
  4. 系统会生成 Bot ID 和 Secret，保存好
  5. 配置可见范围，保存
  6. 在终端安装插件并配置： bash openclaw plugins install @wecom/wecom-openclaw-plugin openclaw configure 填入 Bot ID 和 Secret
  7. 重启： bash openclaw gateway restart

验证

在企业微信里找到你的机器人（工作台 → 智能机器人 → 详情 → 去使用 → 发消息），发一条消息试试！

💡 长连接 vs URL 回调：长连接不需要域名和公网 IP，本地电脑直接用，是推荐方式。URL 回调适合有公网服务器的场景。

第四步：给你的 Agent 一个灵魂（随时可做）

这是最有趣的部分。进入工作区：

cd ~/.openclaw/workspace

IDENTITY.md — 它叫什么

# IDENTITY.md
- **Name:** 你给它起的名字（比如小助、Jarvis、阿呆……）
- **Vibe:** 风格（专业严谨 / 轻松幽默 / 毒舌损友……）
- **Emoji:** 标志 emoji

SOUL.md — 它的性格

定义 Agent 的行为准则： - 回答风格（简洁还是详细？严肃还是幽默？） - 什么时候主动说话 - 处理敏感问题的原则

默认模板已经很好用了，可以在使用中逐渐调整。

USER.md — 关于你

# USER.md
- **Name:** 你的名字
- **Timezone:** Asia/Shanghai
- **Notes:** 你关心什么、做什么工作、有什么偏好

第五步：解锁更多能力（可选进阶）

5.1 安装技能包

# 自我改进（从错误中学习）
openclaw skills install self-improving-agent

# 总结网页/文件/视频
openclaw skills install summarize

# 浏览器自动化
openclaw skills install agent-browser

更多技能：https://clawhub.com

5.2 配置网页搜索

openclaw configure --section web

推荐 Brave Search（免费 2000 次/月）：https://brave.com/search/api/

5.3 使用企微文档 & 智能表格（MCP）

企业微信机器人支持通过 MCP 直接创建文档和智能表格！

  1. 先确保插件是最新版： bash openclaw plugins update wecom-openclaw-plugin
  2. 直接对机器人说："帮我整理一份 XXX 的信息，生成企业微信文档"
  3. 机器人会发送授权链接，点击授权即可
  4. 授权后告诉机器人"授权了"，它就会自动生成文档/智能表格

5.4 设置定时任务

# 例：每天早上 9 点发天气预报
openclaw cron add --schedule "0 9 * * *" --task "查询今天天气并发送给我"

实战经验：我是怎么一步步训练 J 的

以下是 jianjun 训练 J（他的专属 AI Agent）的真实时间线，供你参考。从装好到"能用"大约 2 天，但 Agent 会在使用中持续进化。

Day 0（2026-03-17）—— 装好 + 初次对话

做了什么： - 安装 OpenClaw，跑完 onboard 向导 - 接入企业微信 - Agent 第一次上线，执行了 BOOTSTRAP.md（出生仪式）—— 它会问你"我是谁？你是谁？"

关键操作： - 给它起名叫 J，选了 emoji 🅙 - 告诉它我的身份和工作背景 - 一起讨论它的定位：技术专家 + 社交数据专家（因为我的同事经常问这两类问题） - 把这些写进了 IDENTITY.md

心得：

第一次对话就是"认识"的过程。别急着让它干活，先把"你是谁、我是谁、你要为谁服务"聊清楚。这些信息会写入配置文件，成为它永久的底色。

Day 0 晚上 —— 规划知识体系

做了什么： - 因为主要让它服务同事的咨询，所以规划了一套知识框架： - PERSONA.md — 对外人设（以什么身份回答同事） - KNOWLEDGE.md — 知识库（社交平台数据指标、工具安装步骤等） - SCRIPTS.md — 话术手册（欢迎语、追问模板、排队话术）

心得：

如果你的 Agent 主要服务某个特定领域，提前把知识结构想清楚很重要。不用一次写完，但框架要有。

Day 1（2026-03-18）—— 定规矩 + 调能力

上午：制定安全规则

这是最重要的一步。我写了 4 条全局行为准则到 AGENTS.md：

  1. 高级指令仅限我本人 — 其他人不能让它改配置、装插件、改系统
  2. 隐私信息绝对保密 — 我和它的聊天记录、个人信息，谁问都不给
  3. 对外只做咨询 — 不帮外人跑重度任务（省 token 费）
  4. 宿主机资源敏感 — 不能随便给外人看我电脑上的文件

心得：

一定要在早期就定好安全边界！Agent 默认是"乐于助人"的，如果不设限，别人让它发文件、改配置它可能真的会做。规矩要写进 AGENTS.md，不能只口头说。

下午：调教浏览器操控能力

  • 让它用 AppleScript + Chrome JavaScript 去小红书搜索内容
  • 踩了一堆坑（Chrome 远程调试丢登录态、screencapture 权限问题、SPA 页面 web_fetch 抓不到……）
  • 最终跑通了：AppleScript 控制 Chrome 执行 JS 读取 DOM
  • 把方法和踩坑经验都记录到了 TOOLS.md

心得：

教 Agent 新技能的过程就是"你试 → 它失败 → 你帮它排查 → 它记住"。关键是让它把经验写下来，写到 TOOLS.md 或 memory 里，下次就不会再犯。

晚上：设置定时巡查

  • 创建了 session-patrol 定时任务，每小时巡查所有会话
  • 用便宜模型（Sonnet）跑巡查，省钱
  • 巡查报告保存到 memory，摘要发企业微信给我

心得：

定时任务是让 Agent "活"起来的关键。它不只是你问一句答一句，而是会主动帮你盯着事情。

Day 2（2026-03-19）—— 实战打磨

小红书搜索实战： - 让它搜"Claude 怎么防止被封"，它完整执行了：打开 Chrome → 搜索 → 抓取 15 条结果 → 尝试点进正文（被小红书拦了）→ 从标题归纳出核心结论 - 虽然正文没抓全，但从搜索结果页已经能提炼出有价值的信息

TTS 语音能力： - 让它用男声介绍自己 - 发现 macOS 没有中文男声 → 它自己安装了 edge-tts（微软免费 TTS）→ 用"云希"男声生成了自我介绍音频 - 全程我只说了一句"用男声介绍一下你自己"，剩下的它自己搞定

摄像头拍照（失败案例）： - 让它试着调摄像头拍照 → 因为 macOS TCC 权限限制，后台进程无法获得摄像头授权 - 试了 imagesnap、Swift AVFoundation、App Bundle 包装、Photo Booth 操控……全部失败 - 最终它老实承认"这个做不到"，并给出了替代方案

心得：

失败案例也很有价值。Agent 会把"做不到的事"记住，下次不再浪费时间。同时你也能了解它的能力边界在哪里。

编写操作指南： - 让它把"怎么打造一个像 J 一样的 Agent"写成指南 → 它自己读了 OpenClaw 文档、企业微信官方文档，整合出了完整教程

持续进化中……

Agent 的训练不是一次性的，而是在使用中持续进化：

  • 每次犯错 → 它会记录到 .learnings/ 目录
  • 每次学到新技能 → 更新 TOOLS.md
  • 每天的对话 → 自动记录到 memory/YYYY-MM-DD.md
  • 定期复盘 → 它会把日志里的重要信息提炼到 MEMORY.md（长期记忆）

训练小结：你需要做的核心动作

阶段 你做什么 Agent 做什么
起名+定位 告诉它叫什么、做什么 写入 IDENTITY.md
定规矩 说清楚什么能做什么不能做 写入 AGENTS.md
喂知识 发资料、聊业务背景 写入 KNOWLEDGE.md / 知识库
教技能 "你试试能不能做 XXX" 尝试 → 成功记录到 TOOLS.md / 失败记录到 .learnings
设边界 纠正它："这个不该说/做" 更新 SOUL.md / AGENTS.md
日常使用 正常提需求 从对话中学习，持续积累记忆

💡 核心理念：你不需要"编程"或"训练模型"。你只需要跟它聊天，把重要的事情说清楚，它会自己把这些转化为持久的配置和记忆。就像带一个新同事 —— 前两天多花点时间，后面就越来越顺了。

常见问题

Q: 需要管理员权限才能接入吗？ 不需要！只要管理员开放了「智能机器人」创建权限，普通成员就能自己创建并接入。大企业（>1000人）默认需要管理员开通长连接权限，中小企业默认开放。

Q: 费用大概多少？ - OpenClaw：开源免费 - AI 模型：Claude Sonnet 约 $3/百万 token，日常每天几毛到几块 - 企业微信：免费

Q: 电脑关了还能用吗？ 不能。如果需要 24 小时在线： - Mac 设置为不休眠（推荐） - 或部署到云服务器（腾讯云 Lighthouse 有一键部署模板）

Q: 机器人能在客户群/外部群用吗？ 暂不支持，目前只能在企业内部使用。

Q: 支持哪些消息格式？ 文字、图片、图文混排、语音、视频、文件都支持（需要多模态模型才能识别图片/附件）。

Q: 怎么更新？

npm install -g openclaw@latest        # 更新 OpenClaw
openclaw plugins update wecom-openclaw-plugin  # 更新企微插件
openclaw gateway restart               # 重启
